System under test: macOS (darwin/arm64) or Linux (x86_64/arm64). Commands assume a POSIX shell (bash or zsh). Windows WSL should also work for most scenarios — any Windows-specific deltas will be called out inline.
Project root: /absolute/path/to/your/project — the Orchestrator will replace this placeholder with the real path before dispatching.
Language runtime: python 3.12.x inside .venv/ at the project root (OR the equivalent for your stack — Node 20.x, Rust 1.75, Go 1.21, etc.; the scenarios below name the exact runtime they exercise).
Required tools: python, pip, git, jq. Optional: sqlite3 (scenarios 4, 7), docker (scenario 13 only).

One-time setup before any scenario:
cd "/absolute/path/to/your/project"
source .venv/bin/activate
python --version # must print 3.12.x

Every scenario below assumes:
1. You are in the project-root terminal above.
2. The .venv is active (your prompt should show (.venv)).
3. If a scenario changes directory, it says so and returns you to the project root before ending.
4. If a scenario mutates files, it includes explicit cleanup + verification.

How to run a scenario: click "details" on the scenario card, copy the numbered commands into the terminal, compare your output to the "Expected" block, then click Pass / Fail / Skip. Use the Notes field for anything surprising or out of scope.
